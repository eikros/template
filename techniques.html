Techniques

Before getting started with Template, there are a few techniques you should be aware of.

Inline blocks

Template uses inline-block extensively to create horizontal layouts. In fact there are no floats in Template. Inline blocks have advantages over floats:

  • They stay in the flow, so their parents always 'clear'
  • It is possible to write them in arbitrary content order
  • Inline block layouts have direct analogues with parts of the upcoming box-layout CSS3 module

Activatable elements

Popdowns, dropdowns, tabs and slides are ui elements with two states. When they are visible, they have the class active. When they are not, they don't. They share a common way to trigger active state.

Firstly, they are activated when the user clicks on a link that refers to them by id:

{% highlight html %}Open dropdown{% endhighlight %}

Secondly, they can also be activated and deactivated programmatically:

{% highlight js %}jQuery('#id_of_dropdown').trigger('activate');{% endhighlight %} {% highlight js %}jQuery('#id_of_dropdown').trigger('deactivate');{% endhighlight %}

Creating a popdown, dropdown, set of tabs or a slide is as easy as adding a class to an element, then linking to it by id. Links that refer to popdowns, dropdowns, tabs and slides are also given the class active, making it easy to create stateful buttons. Transitions to and from active state are defined using CSS transitions.

.tab and .slide

Tabs and slides are functionally identical. Two classes are provided for convenience, as tabs and slides tend to be styled very differently. Where we refer to 'tabs' below, assume that we also mean slides.

In any group of tabs, only one can be active at any one time. Normally, all .tab or .slide siblings of the tab or slide currently being activated automatically become a group, but to be more explicit, the grouping can be overridden with a data-selector attribute containing a selector for all other tabs in the group.
