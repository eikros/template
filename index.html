About template

Template is essentially a library of classes, some of them with javascript behaviours attached, intended for use as base classes in OOCSS-style development, but also useable or as mixins in a SASS or LESS environment.

Template concentrates on layout techniques. It gives you flexible columns with arbitrary content order, thumbs with hidden text, horizontal lists, popups that adapt to the size of their contents, drop-downs, tabs and slides, layers that fill the window, and more. It does not give you much colour. This is deliberate. Most sites and webapps require the same layout techniques; few use the same colours.

It is not a huge library, it really is just a starting point, but to make the most of it, it is worth understanding the CSS. I'll declare the principal gotcha up-front: Template uses inline-blocks, and never floats, for horizontal layout. Inline blocks require spaceless HTML. More on that at techniques.html#inlineblocks.

.wrap

The .wrap class is very basic. It is there to act as a position parent. It also has margin-left: auto; margin-rght: auto;, so that when it is given explicit width it centres itself horizontally.

.tab

In any group of tabs, only one can be active at any one time. Normally, all .tab siblings of the tab or slide currently being activated automatically become a group, but it is possible to explicitly create groups.

.slide

Slides are functionally identical to tabs – only one can be active at any one time. They differ in default style, where slides have the popular slide-in-from-the-side animation when they are activated. Slides also react to the special #prev and #next links.

Note that slides don't by default have a parent with overflow: hidden;. Usually you want one of those – but not always.
